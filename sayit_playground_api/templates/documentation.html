sayIt API Documentation

Complete guide to our REST API endpoints and authentication

Getting Started

Welcome to the sayIt Playground API documentation. This guide will help you seamlessly integrate fun games and trivia into your applications. Our RESTful API uses JSON for data exchange, making it easy and efficient to add engaging features to your projects.

The Playground API provides a wide variety of interactive games, trivia questions, and entertainment endpoints, allowing you to quickly add playful and engaging experiences to your websites and apps with simple HTTP requests.

Your API Key

You need an API key to use the API - this is a unique key that identifies your requests. They're free while you're in development.

An API key can only be assigned to you after you have created an account. If you are logged in, you will see your live API key in all the examples.

Base URL

All API requests should be made to the following base URL: 

https://api.example.com/v1/

All API responses are returned in JSON format with UTF-8 encoding.

Authentication

Authentication is handled with a simple API key. They're free while you are in development, and you can get one here:

API Key Authentication

Via Querystring 
GET https://newsapi.org/v2/everything?q=keyword&apiKey=API_KEY
Via X-Api-Key HTTP header 
X-Api-Key: API_KEY
Via Authorization HTTP header 
Authorization: API_KEY

An Via the Authorization HTTP header. Including Bearer is optional, and be sure not to base 64 encode it like you may have seen in other authentication tutorials

If you don't append your API key correctly, or your API key is invalid, you will receive a 401 - Unauthorized HTTP error.

Endpoints

Playground API has 3 endpoints:

All Game Types /api/v1/all-game-types

This returns all the types of games that are avialable on playground at the moment

Game By Type /api/v1/get-random

Returns a game according to types specified in the params of the API

Random Game Type /api/v1/get-by-type

Returns a random game type without specification

Query Parameters

Parameter Type Description Required
api_key str Your unique API Key Yes
game_type str Type of Game Yes
category str Kind of game No
limit int Maximum Number of games you want returned No

Error Handling

If you make a bad request we'll let you know by returning a relevant HTTP status code along with more details in the body.

HTTP Status Codes

200 OK - Request successful

201 Created - Resource created successfully

400 Bad Request - Invalid request parameters

401 Unauthorized - Authentication required

403 Forbidden - Insufficient permissions

404 Not Found - Resource not found

500 Internal Server Error - Server error

Error Response Format

All error responses follow a consistent format: 


{
    "error": {
    "code": "INVALID_CREDENTIALS",
    "message": "The provided credentials are invalid",
    "details": {
    "field": "email",
    "issue": "Email format is invalid"
    },
    "timestamp": "2024-07-07T14:30:00Z"
    }
}

Common Error Codes

Code Description HTTP Status
INVALID_CREDENTIALS Invalid username or password 401
INSUFFICIENT_PERMISSIONS User lacks required permissions 403
RESOURCE_NOT_FOUND Requested resource does not exist 404
RATE_LIMIT_EXCEEDED Too many requests in a given time period 429
